Machine Learning Models

This repository includes machine learning modules for the SDRT project. It contains 2 main directories:

uji - ML techniques for positioning prediction of tags in a multi building and floor environment. It is based on the UJIIndoorLoc database, created in the Universitat Jaume I. More info here: https://archive.ics.uci.edu/ml/datasets/ujiindoorloc

simulator_predictor - ML techniques based on the SDRT simulator. It contains python scripts for model training, stored in the corresponding location directory and a python script "predictor.py" that contains a MQTT agent responsible for communicating with the simulator and predict the location of assets based on their rssi values and the previously trained ml models.

By default, the predictor.py script will make predictions based on the models stored in the "somos_saude" dir but this could be changed by running with --models "name of another folder": python3 predictor.py --models it

Model Training: The train.sh script was created as an easier and faster way to train all the models. To use it, there must be a dataset which name starts with "rssi_antennas" in the same dir as the script, where shouldn't be stored any important model with the same name as the new ones, otherwise they will be overwritten. The dir which the models will be moved into is the "somos_saude" by default, but it can be changed in the script.

Main features:

  • MQTT client for communication (predictor)
  • Model trainning
  • Asset location prediction based on stored models

It implies the creation of a virtual environment (venv) whose dependencies come in the requirements.txt file.

Usefull commands:

  • install venv package: > sudo apt-get install python3-venv
  • remove .venv directory: > rm -Rfv .venv
  • create .venv directory: > python3 -m venv .venv
  • activate .venv: > source .venv/bin/activate
  • install dependencies on .venv from requirements.txt: > pip install -r requirements.txt
  • generate documentation using pdoc: > pdoc --math -d google -o docs predictor.py

 37def degrees_to_metres(deg: float) -> float:
 38    """
 39    Coordinates degrees to meters conversor using the following formula:
 40    $$
 41    meters=\\frac{2 \\times \\pi \\times earthRadius \\times degrees}{360}
 42    $$
 43
 44    Args:
 45        deg (float): coordinate in degrees
 46
 47    Returns:
 48        float: coordinate in meters
 49    """
 50    return (2 * math.pi * 6371000 * deg) / 360

 52def metres_to_degrees(metres: float) -> float:
 53    """
 54    Coordinates meters to degrees conversor using the following formula:
 55    $$
 56    degrees=\\frac{360 \\times meters}{2 \\times \\pi \\times earthRadius}
 57    $$
 58
 59    Args:
 60        metres (float): coordinate in degrees
 61
 62    Returns:
 63        float: coordinate in degrees
 64    """
 65    return (metres * 360) / (2 * math.pi * 6371000)

 85def min_max_scale(val: float, min_val: float, max_val: float) -> float:
 86    """
 87    Min max scaler.
 88
 89    Args:
 90        val (float): value to be scaled
 91        min_val (float): minimum value
 92        max_val (float): maximum value
 93
 94    Returns:
 95        float: min_max_scaler
 96    """
 97    val = (val-min_val)/(max_val-min_val)
 98    return val

100def reverse_min_max_scale(val: float, min_val: float, max_val: float) -> float:
101    """
102    Min max reverse scaler.
103
104    Args:
105        val (float): value to be unscaled
106        min_val (float): minimum value
107        max_val (float): maximum value
108
109    Returns:
110        float: reverse_min_max_scaler
111    """
112    val = (val * (max_val-min_val))+min_val
113    return val

115def get_avg_rssi(d: list) -> dict:
116    """
117    Makes an average of the rssi values in each iteration.
118
119    Args:
120        d (list): list of rssi values
121
122    Returns:
123        dict: average rssi values
124    """
125    return_dict = {}
126    for iter in d:
127        for col in range(0, len(iter)):
128            if col in return_dict:
129                return_dict[col] += float(iter[col])
130            else:
131                return_dict[col] = float(iter[col])
132    for col in return_dict:
133        return_dict[col] = return_dict[col] / len(d)
134    return return_dict

136def scale_sd(val:list) -> list:
137    """
138    Scales using the min and max coordinates.
139
140    Args:
141        val (list): coordinates to be scaled
142
143    Returns:
144        float: scaled coordinates
145    """
146    for i in range(len(val)):
147        val[i][0] = min_max_scale(val[i][0], lng_min, lng_max)
148        val[i][1] = min_max_scale(val[i][1], lat_min, lat_max)
149    return val

151def reverse_scale_val(val:list) -> list:
152    """
153    Unscales using the min and max coordinates.
154
155    Args:
156        val (list): coordinates to be unscaled
157
158    Returns:
159        float: real coordinates
160    """
161    val[0][0] = reverse_min_max_scale(val[0][0], lng_min, lng_max)
162    val[0][1] = reverse_min_max_scale(val[0][1], lat_min, lat_max)
163    return val

165def get_coord_from_dist(dist: float, antenna_coord: list, antenna_angle: float) -> tuple:
166    """
167    Estimates the asset coordinates using data regarding the distance to an antenna, its position and angle.
168
169    Args:
170        dist (float): distance from the tag to the antenna
171        antenna_coord (list): coordinates of the antenna
172        antenna_angle (float): angle of direction of the antenna
173
174    Returns:
175        tuple: asset coordinates
176    """
177    rad_angle = math.radians(antenna_angle)
178    lng = degrees_to_metres(antenna_coord[0]) - math.sin(rad_angle) * dist
179    lat = degrees_to_metres(antenna_coord[1]) + math.cos(rad_angle) * dist
180    return lng, lat

189        anchors_activ (list): activations of the anchors

198                for pwr_i in range(len(antenna)):
